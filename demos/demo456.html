Scrawl multiFilters - matrix

Extensions: images, filters, animation

The matrix filter accepts four attributes: width, height, offsetX and offsetY. The offsets are relative to the current pixel during the calculations, so for a 5 x 5 matrix centred on the current pixel, the offset values should be -2. The number of items in the weights array must match the volume of the matrix (width * height). The filter below uses the following values:

  • width: 5,
  • height: 5,
  • offsetX: -2,
  • offsetY: -2,
  • weights: [-1, 0, 0, 0, 0, 0, -2, 0, 0, 0, 0, 0, 6, 0, 0, 0, 0, 0, -2, 0, 0, 0, 0, 0, -1]

This filter caches precalculated data in the multiFilter cache - MultiFilter.cache.matrix - when updating the filter's attributes dynamically this cache needs to be set to false so the filter knows it needs recalculating.
