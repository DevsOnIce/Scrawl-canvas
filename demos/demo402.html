Scrawl multiFilters - filters which take a level argument

Extensions: images, multifilters, animation

The 'default' filter does not apply any filter engines to the parrot Picture entity. It does, however, make the entity use the multiFilter stamp code rather than the normal code. The Wheel entity, which is stamped on the cell before the parrot, has no multiFilter set on it.

The other filters in this demo all accept an additional 'level' attribute:

  • For the brightness filter, this ranges from 0+ with 0 = black, 1 = normal and 1+ = over-bright
  • For the saturation filter, this ranges from 0+ with 0 = mid-gray, 1 = normal and 1+ = over-saturated
  • For the threshold filter, this ranges from 0-1, where the level indicates at which point a darker pixel becomes black and a lighter one white
